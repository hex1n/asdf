# Adapter · RPC

二进制/注册中心型远程调用（SOFARPC、Dubbo、gRPC…）。不走 URL path + HTTP method，调用按服务接口经注册中心/网关定位。

**本 adapter 给的是探测办法，不写死任何单一项目的值**——基址 / 信封字段 / 包装类 / 鉴权注解 / 校验 util / 目录布局都在目标项目**现读**（探不到再问）。框架注解是 grep 锚点，不是默认取值。

## 接口发现
- 服务**接口类** + **实现类**；接口每个方法 = 一次远程调用。
- 实现类的框架标记（grep 定位）：SOFARPC `@SofaService`、Dubbo `@DubboService`、gRPC 由 `.proto` 生成 stub。
- 命名/文件模式按项目惯例 grep（常见 `*Facade.java` / `*Service.java` + 对应 `*Impl.java`）。
- 接口与实现的相对位置按**实际文件位置**确定，不假定固定目录。

## 地址与信封
- **操作标识**：方法对外的操作名/路由键——若接口方法带**网关派发注解**（注解值承载对外操作标识），取其值；否则用「服务接口全名 + 方法名」。grep 接口确认有无此类注解。
- **基址**：探 `application.yml`/`.properties` / 网关配置 / README；探不到再问。
- **信封**（若经统一网关派发）：
  - 入参信封 = 项目的**派发入参对象**字段（承载操作标识 + 业务参数的那个外层对象）；
  - 响应信封 = **所有接口返回共用的那层外壳类**——看任一接口实现的返回类型及其公共基类，读其字段（典型含「是否成功 / 错误码 / 错误信息 / data / 链路号」之类）。
  - **构件名与字段名各项目不同，必须现读，不照搬别项目。**
- 请求表**无「位置」列**——字段都在入参对象内。

## 认证（严格按代码现存声明，只读不改；疑似缺认证只提示，不擅自加）
- 找**调用前校验身份/令牌/权限**的声明：方法或类上的鉴权注解、AOP 切面、网关 Filter/拦截器——名字各项目不同，按职责识别。
- 有鉴权 → 🔐（注明机制/凭证位置/所需角色）；确认放行 → 🌐；查不到声明 → 🔐? 并提示「未见鉴权声明，需确认」，不擅自下结论。

## 必填
**以实际为准**：入口处对请求字段的**非空/合法性断言**（断言工具调用）> 标准校验注解（`@NotNull`/`@Valid` 等）> 默认 N。业务必传但未声明的标 N，备注「业务必传」。

## ID 类型
以报文（接口签名/DTO）声明类型为准；留意项目惯例（如「报文 String、DAL Long」——若存在，按报文类型写）。
